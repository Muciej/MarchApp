Z plikow w tym folderze można odtworzyć bazę danych dla naszej aplikacji
wraz z podstawowymi danymi.

Kolejność:
baza_biegow_przejalowych_init.sql -> inicjuje główną bazę przechowującą informacje o eventach
test_event_database_init.sql 0 -> tworzy bazę danych dla przykładowego eventu "test_event"
fill_test_data.sql -> wypełnia utworzone bazy testowymi danymi
database_test_accounts.sql -> tworzy testowe konta baz danych dla poszczególnych biegów oraz konto pozwalające wyświetlać obecne biegi
